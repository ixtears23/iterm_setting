# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 프로젝트 개요

iTerm2 파워 유저 세팅을 자동화하는 프로젝트. 이미 iTerm2가 설정되어 있더라도 초기화 후 새로 세팅할 수 있도록 구성한다.

## Git 커밋 규칙

### 커밋 메시지 형식

```
#이슈번호 [타입] 제목 (구체적으로, 한국어)

- 변경 사유 또는 배경
- 구체적인 변경 내용
- 영향 범위 또는 참고 사항
```

### 타입 목록

| 타입 | 설명 |
|------|------|
| `기능` | 새로운 기능 추가 |
| `수정` | 버그 수정 |
| `핫픽스` | 긴급 수정 |
| `리팩토링` | 동작 변경 없이 코드 구조 개선 |
| `문서` | 문서 작성 또는 수정 |
| `설계` | 아키텍처 또는 설계 변경 |
| `스타일` | 포맷팅, 세미콜론 등 코드 스타일 변경 (동작 무관) |
| `설정` | 설정 파일 추가 또는 변경 |
| `삭제` | 불필요한 코드 또는 파일 제거 |

### 커밋 원칙

- **커밋은 짧고 자주 한다.** 하나의 커밋에 하나의 변경 단위만 포함한다.
- **제목은 구체적으로 적는다.** 제목만 보고도 무엇이 변경되었는지 알 수 있어야 한다.
- **본문은 상세히 적는다.** 왜 변경했는지, 무엇이 바뀌었는지, 영향 범위는 어디인지 기술한다.
- **이슈 번호가 없으면 `#0`을 사용한다.**
- **커밋할 타이밍이 되면 사용자에게 먼저 물어본다.** 자동으로 커밋하지 않는다.

### 커밋 메시지 예시

```
#12 [기능] Oh My Zsh 자동 설치 스크립트 추가

- iTerm2 초기 세팅 시 Oh My Zsh가 없으면 자동 설치
- 이미 설치된 경우 건너뛰는 로직 포함
- install.sh에서 호출되는 구조
```

```
#0 [문서] CLAUDE.md에 커밋 규칙 및 프로젝트 개요 작성

- Git 커밋 메시지 형식과 타입 목록 정의
- 프로젝트 목적 및 작업 규칙 기술
- HANDOFF.md, CHANGELOG.md 운영 방침 추가
```

## 파일 운영 규칙

### HANDOFF.md

세션이 종료되기 전에 반드시 `HANDOFF.md`를 업데이트한다. 새 세션에서 이 파일을 읽고 이전 작업을 이어갈 수 있어야 한다.

### CHANGELOG.md

의미 있는 변경이 있을 때마다 `CHANGELOG.md`에 기록한다. 날짜 기준으로 최신이 위에 온다.

## Allowed Commands
